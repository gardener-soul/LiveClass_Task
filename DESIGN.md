# LiveKlass 수강 신청 프로젝트: UI/UX 디자인 및 설계 문서

본 문서는 LiveKlass의 Mock 페이지 구조와 Frontend 과제 요구사항을 기반으로 작성된 UI/UX 디자인 및 컴포넌트 설계 문서입니다.

---

## 1. 강의 선택 창 UI (`/classes`)
LiveKlass의 메인 카탈로그 화면으로, 사용자가 원하는 강의를 쉽게 탐색하고 선택할 수 있는 화면입니다.

### Layout & Structure
- **Header**: 로고 및 메인 GNB 네비게이션
- **Category Navigation**: "전체", "개발", "디자인", "마케팅", "비즈니스" 등 API 카테고리(`Course.category`)와 연동되는 가로형 탭 메뉴.
- **Search & Filter Bar**:
  - 최신순, 인기순 등의 정렬 드롭다운
  - 텍스트 검색을 위한 돋보기 아이콘이 포함된 Search Input
- **Main Content**: 3~4열(모바일의 경우 1열) Grid 레이아웃의 강의 카드 목록

### UI Components
- **Course Card**:
  - **Thumbnail**: 상단 영역 이미지 배치 및 상태 뱃지(모집중, 마감 임박 등) 오버레이
  - **Metadata**: 수강 기간, 최대 인원, 강사명 등 아이콘과 함께 표시
  - **Title**: 최대 2줄 텍스트 (말줄임표 처리)
  - **Price**: 원래 가격(취소선)과 최종 가격 강조 처리

### UX Requirements (과제 요구사항 반영)
- **빈 상태 (Empty State)**: 필터링이나 검색 결과가 비어있을 경우, "조건에 맞는 강의가 없습니다"라는 명확한 문구와 일러스트 제공.
- **로딩 상태 (Loading State)**: API(`/api/courses`) 페칭 중 사용자 경험을 위해 카드 형태의 **Skeleton UI** 노출.

## 2. 세부 강의 정보 UI (`/classes/:id`)
강의의 상세 정보를 제공하고 수강 신청을 유도하는 랜딩 페이지 성격의 화면입니다.

### Layout & Structure
- **Split Layout (2 Column)**: 데스크톱 환경에서는 좌측 메인 콘텐츠와 우측 Sticky 사이드바로 분리하여 스크롤 시에도 결제 및 신청 액션이 항상 보이도록 설계.

### UI Components
- **Left Column (Main Content)**:
  - Hero Image (강의 배너)
  - 네비게이션 탭 (클래스 소개, 커리큘럼, 수강 후기, 커뮤니티 등)
  - 모집 기간(`startDate` ~ `endDate`) 및 상세 설명 구역
- **Right Column (Sticky Sidebar)**:
  - 강의 타이틀 및 요약 정보
  - 최종 수강료 강조
  - **Primary Action Button**: 커다란 "클래스 신청" 버튼. (클릭 시 3번 항목의 신청 폼으로 라우팅)

### UX Requirements (과제 요구사항 반영)
- **정원 마감 처리**: API 데이터 기반으로 `currentEnrollment >= maxCapacity`인 경우, "클래스 신청" 버튼을 회색으로 비활성화(Disabled)하고 텍스트를 "모집 마감"으로 변경하여 진입을 차단.

## 3. 강의 신청 UI (Multi-step Form: 1~2단계)
가장 핵심이 되는 과제 요구사항으로, 복잡한 신청 과정을 분리하여 이탈률을 낮추는 **Stepper** 형태의 폼 구조입니다.

### Layout & Structure
- **Step Indicator**: 화면 상단에 현재 진행 상황을 시각적으로 표시. (예: `1. 정보 입력` → `2. 확인 및 제출`)
- **Centered Layout**: 사용자가 폼 입력에 집중할 수 있도록 원-컬럼(Single Column)의 좁고 중앙 정렬된 카드형 레이아웃 사용.

### 정보 입력 스텝 (핵심 분기 처리)
- **Type Toggle**: [개인 신청] / [단체 신청] 전환을 위한 세그먼트 컨트롤(Segmented Control) 또는 라디오 버튼.

**[개인 신청 필드]**
- 이름, 이메일, 연락처 (필수)
- 지원 동기 (선택)

**[단체 신청 필드]**
- 신청자 정보 (이름, 이메일, 연락처)
- 단체 정보 (단체명, 총 인원수, 담당자명)
- **참가자 명단 폼**: 입력된 '총 인원수'에 맞춰 이름/이메일을 입력할 수 있는 동적 폼 배열. 인원이 많을 경우를 대비해 아코디언 형태나 엑셀 업로드 방식 고려 (UX 가산점 요소).

### UX Requirements (과제 평가 기준 반영)
- **조건부 필드 데이터 처리 (중요)**: 단체 신청으로 작성하던 중 '개인 신청'으로 타입을 변경할 경우, 입력된 단체 정보(명단 등)가 방치되지 않도록 **초기화 확인 대화상자(Confirm Dialog)** 노출 및 데이터 클렌징 처리.
- **유효성 검증 시점**: 폼 제출 시에만 검증하는 것이 아니라, 필드 입력 후 포커스를 잃을 때(`onBlur`) 개별 검증 수행.
- **에러 시각화**: 에러 발생 시 텍스트만 표시하는 것이 아니라, 해당 Input 필드의 테두리를 붉은색으로 변경하고 에러 메시지를 하단에 명확히 표기.
- **이탈 방지 / 데이터 복구**:
  - 작성 중 브라우저 뒤로가기/닫기 시 "작성 중인 내용이 사라집니다" 브라우저 Alert 연동.
  - 새로고침 시에도 입력 데이터 유지를 위한 `localStorage` 기반 임시 저장 연동 (선택 구현 사항).

## 4. 확인 및 제출 폼 (Multi-step Form: 3단계)
입력한 모든 정보를 마지막으로 점검하고 실제 서버 제출(`POST /api/enrollments`)을 수행하는 화면입니다.

### Layout & Structure
- **Summary Card**: 사용자가 입력한 신청 유형(개인/단체), 강의 정보, 입력한 인적사항 및 명단을 읽기 쉬운 표나 리스트 형태로 정리하여 노출.
- **Edit Links**: 각 요약 정보 블록의 우측 상단에 "수정" 링크를 배치하여 클릭 시 직전 스텝의 해당 필드로 이동하도록 지원.
- **Terms Agreement**: 서비스 이용약관 및 개인정보 처리방침 동의 체크박스 (LiveKlass의 커스텀 체크박스 스타일 차용).
- **Submit Button**: 하단 고정 형태의 넓은 "최종 신청하기" 버튼.

### UX Requirements (과제 평가 기준 반영)
- **유효성 에러 포커싱 (Scroll to Error)**: 최종 제출 버튼 클릭 시 누락된 항목이나 에러가 발생한 필드가 있다면, 화면을 해당 위치로 **부드럽게 스크롤(Smooth Scroll)** 시키고 필드를 시각적으로 강조(Blinking 등).
- **중복 제출 방지**: 제출 버튼 연타로 인한 API 중복 호출을 막기 위해, 클릭 즉시 버튼을 로딩 상태(Spinner 아이콘)로 변경하고 비활성화 처리.
- **제출 실패 예외 처리 (Error Handling)**:
  - 서버 측 비즈니스 에러 (`COURSE_FULL`: 정원 초과, `DUPLICATE_ENROLLMENT`: 이미 신청된 강의) 발생 시, 단순 토스트(Toast) 메시지가 아닌 폼 상단에 명확한 **에러 Alert 박스** 렌더링.
  - 에러가 발생하더라도 사용자가 입력한 데이터 상태는 모두 유지하여, 수정 후 재시도가 가능하도록 설계.
