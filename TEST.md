## Mock API 에러 트리거

Mock API는 실제 서버 없이 에러 시나리오를 재현할 수 있도록 **특정 입력값 기반 트리거**를 사용합니다.

### COURSE_FULL — 제출 시 서버 정원 초과 에러

| 항목 | 값 |
|------|-----|
| 트리거 강의 ID | `course-dev-02` |
| 강의명 | TypeScript 심화 과정 (마감 임박 배지 표시) |
| 트리거 시점 | Step 3 제출 |

> 이 강의는 Step 1에서 선택 가능하지만, 제출 시 서버에서 `COURSE_FULL` 에러를 반환합니다.

**기대 동작**
- Step 3 제출 후 폼 상단 에러 Alert: `"해당 강의의 정원이 마감되었습니다"`
- 1단계로 이동 버튼 제공
- 입력 데이터 전체 유지

---

### DUPLICATE_ENROLLMENT — 이미 신청된 강의 에러

| 항목 | 값 |
|------|-----|
| 트리거 이메일 | `duplicate@test.com` |
| 트리거 시점 | Step 3 제출 |

**기대 동작**
- 폼 상단 에러 Alert: `"이미 신청하신 강의입니다"`
- 입력 데이터 전체 유지

---

### INVALID_INPUT — 서버 측 입력값 검증 실패 에러

| 항목 | 값 |
|------|-----|
| 트리거 조건 | 신청자 이름을 `invalid`로 시작 (예: `invalid김철수`) |
| 트리거 시점 | Step 3 제출 |

**기대 동작**
- `applicant.name` 필드에 에러 매핑: `"허용되지 않는 이름입니다"`
- Step 2로 이동 후 에러 필드 확인 가능
- 입력 데이터 전체 유지

## 수동 테스트 시나리오

### 성공 케이스

**개인 신청 정상 완료**
1. Step 1: 아래 "일반 강의" 목록에서 임의 강의 선택, 개인 신청 선택
2. Step 2: 에러 트리거 조건에 해당하지 않는 이름·이메일 입력
3. Step 3: 약관 동의 후 제출
4. **기대 결과**: 신청 완료 화면 (enrollmentId, enrolledAt 표시)

**단체 신청 정상 완료**
1. Step 1: 잔여석 ≥ 2인 강의 선택, 단체 신청 선택
2. Step 2: 단체명·인원수(2~10)·참가자 명단 입력 (이메일 중복 없이)
3. Step 3: 약관 동의 후 제출
4. **기대 결과**: 신청 완료 화면

---

### 클라이언트 유효성 검증

**Step 2 — 공통 필드**

| 필드 | 잘못된 입력 예시 | 에러 메시지 |
|------|----------------|-----------|
| 이름 | `김` (1자) | 이름은 2자 이상이어야 합니다 |
| 이름 | `김철수1` (숫자 포함) | 한글 또는 영문만 입력 가능합니다 |
| 이메일 | `test@` | 올바른 이메일 형식이 아닙니다 |
| 전화번호 | `01012345678` (하이픈 없음) | 올바른 전화번호 형식이 아닙니다 (예: 010-1234-5678) |
| 수강 동기 | 301자 이상 | 수강 동기는 300자 이내로 입력해주세요 |

**Step 2 단체 신청 — 인원수 및 참가자**

| 케이스 | 기대 동작 |
|--------|----------|
| 인원수 1 입력 | `신청 인원수는 2~10명이어야 합니다` 에러 |
| 인원수 11 입력 | `신청 인원수는 2~10명이어야 합니다` 에러 |
| 참가자 이메일 중복 | 중복된 인덱스 필드에 `중복된 이메일입니다` 에러 |
| 참가자 이메일 = 신청자 이메일 | `신청자 이메일과 중복됩니다` 에러 |

**Step 3 — 약관 동의 미체크**

| 케이스 | 기대 동작 |
|--------|----------|
| 약관 미동의 후 제출 | `이용약관에 동의해주세요` 에러, 제출 차단 |

### 정원 마감 강의 (Step 1 UI)

아래 강의는 Mock 데이터에서 `currentEnrollment === maxCapacity`로 설정되어 있습니다.

| 강의명 | 카테고리 | 기대 동작 |
|--------|---------|----------|
| Node.js + NestJS 백엔드 개발 | development | disabled + "정원 마감" 배지, 클릭 불가 |
| 콘텐츠 마케팅 전략 | marketing | disabled + "정원 마감" 배지, 클릭 불가 |

---

### 조건부 필드 전환

| 케이스 | 기대 동작 |
|--------|----------|
| 개인 → 단체 전환 | 확인 없이 즉시 단체 필드 표시 |
| 단체 정보 입력 후 개인으로 전환 | 확인 다이얼로그 표시 |
| 다이얼로그 확인 클릭 | 단체 데이터 초기화, 개인 신청 필드로 전환 |
| 다이얼로그 취소 클릭 | 단체 신청 상태 유지, 기존 입력 데이터 보존 |
| 잔여석 < 2인 강의에서 단체 신청 선택 | 확인 없이 개인 신청으로 강제 전환 + 안내 |

### 중복 제출 방지

1. Step 3 제출 버튼 클릭 직후 버튼이 로딩 상태로 변경되는지 확인
2. 로딩 중 버튼 재클릭이 차단되는지 확인 (`disabled` 상태)

---

### localStorage 복구

1. Step 2까지 진입 후 일부 정보 입력
2. 브라우저 새로고침
3. **기대 결과**: 상단에 "이전에 작성하던 내용을 불러왔습니다." 배너 표시, 입력값 복구
4. 배너의 초기화 버튼 클릭 시 폼 초기화 확인
