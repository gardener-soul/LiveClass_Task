# 다단계 수강 신청 폼

라이브클래스 온라인 교육 플랫폼의 수강 신청 흐름을 다단계 폼으로 구현한 과제입니다.

---

## 프로젝트 개요

수강생이 강의를 선택하고 개인/단체 신청 정보를 입력하여 수강 신청을 완료하는 3단계 폼입니다.

- **1단계**: 강의 선택 + 신청 유형(개인/단체) 선택
- **2단계**: 수강생 정보 입력 (신청 유형에 따라 조건부 필드 표시)
- **3단계**: 전체 내용 확인 및 제출

## 요구사항 해석 및 가정

- **시간 충돌(조건부 필드 전환)**: 단체→개인 신청 유형 변경 시 단체 관련 데이터(단체명, 참가자 명단 등)를 초기화합니다. 사용자가 이미 단체 정보를 입력한 상태라면 확인 다이얼로그를 표시합니다.
- **참가자 명단 이메일 중복**: 참가자 명단 내 동일 이메일 중복 입력을 클라이언트에서 즉시 차단합니다.
- **정원 초과 강의**: 강의 목록에서 `currentEnrollment >= maxCapacity`인 강의는 선택 불가 처리하고, 거의 찬 강의(90% 이상)는 시각적으로 표시합니다.
- **유효성 검증 시점**: 각 스텝 전환 시 해당 스텝을 검증하고, 개별 필드는 blur 이벤트에서 검증합니다.

## 설계 결정과 이유

**폼 상태관리 — React Hook Form + 상위 state**

스텝 간 데이터 공유는 `App.tsx`의 상위 state로 관리하고, 각 스텝 내부 폼 상태는 React Hook Form이 담당합니다. Context를 별도로 쓰지 않은 이유는 현재 스텝 구조(3단계 선형 흐름)에서 prop drilling 깊이가 얕아 불필요한 복잡도를 피하기 위해서입니다.

**타입 설계 — discriminated union**

개인/단체 신청 데이터를 `type: "personal" | "group"` 기반의 discriminated union으로 분리하여, 단체 신청 시에만 `group` 필드가 존재함을 타입 레벨에서 보장합니다.

**Mock API**

`src/mocks/`에 Mock 데이터와 fetch 함수를 직접 구현했습니다. 실제 API 교체 시 Mock 함수만 실 API 호출로 변경하면 되도록 인터페이스를 동일하게 유지했습니다.
